# AetherFlow WMS Plan

## 协作方式

- 大方向只看阶段：阶段一 / 阶段二 / 阶段三。
- 日常开发由“代办池”驱动，不在阶段描述里展开实现细节。
- 代办状态统一标记：`[TODO]`、`[DOING]`、`[DONE]`。
- 规则：代办不可删除；完成后改状态并补一行“完成说明”。

---

## 阶段总览

### 阶段一（MVP 闭环）

- 建立 WMS 核心业务闭环：主数据 -> 入库 -> 出库 -> 库存查询。
- 完成最小可用权限能力（登录 + 基础鉴权）。
- 核心目标：业务跑通、数据一致、可稳定演示。

### 阶段二（可配置与可复制）

- 补齐字典、参数、流程配置能力（减少硬编码）。
- 强化主数据联动、规则校验、异常处理。
- 核心目标：不同场景通过配置复用，不靠代码分叉。

### 阶段三（可视化与扩展）

- 引入空间模型与可视化能力（含 2D/3D 展示准备）。
- 评估与 ERP/WCS/外部系统协同能力。
- 核心目标：从“能用”进化到“可扩展、可产品化”。

### [TODO] TODO-A001 区域模型引入（`warehouse -> area -> location`）

- 背景：
  当前只有仓库与库位，缺少区域层，不利于流程分区和后续空间展示扩展。
- 目标：
  在不破坏库存核心口径（`warehouse + location + material`）前提下，引入 `area`。
- 待办：
  1. 新增 `area` 主数据（建议字段：`id`、`warehouse_id`、`area_code`、`area_name`、`area_type`、`status`、`remark`）。
  2. `location` 增加 `area_id`，库位必须隶属于区域。
  3. 增加约束：区域归属仓库唯一、库位归属区域唯一。
  4. 查询支持按 `warehouse_id + area_id` 联合筛选。
  5. 前端联动口径：仓库 -> 区域 -> 库位。
- 验收标准：
  1. 区域可增改查停用。
  2. 库位能稳定关联区域并正确回显。
  3. 单据页联动不出现跨仓错误数据。

- 目标：

  5. 不调整库存唯一维度，不将空间字段纳入库存唯一键。
- 验收标准：
  1. 空间字段可维护、可查询、可回显。
  2. 不影响现有入库/出库/库存主流程。
  3. 为后续 3D 映射保留完整语义。

### [TODO] TODO-A003 主数据下拉与缓存策略落地（查多改少）

- 背景：
  仓库/区域/库位/物料是高频查询、低频修改，业务单据应以“选择”为主。
- 目标：
  减少自由输入导致的脏数据，提高录入效率和一致性。
- 待办：
  1. 后端新增轻量选项接口（仅 `id/code/name/status`），与分页列表接口分离。
  2. 前端下拉统一走选项接口，不直接依赖分页列表接口。
  3. 联动规则：
     - 仓库变化：清空并重载区域/库位。
     - 区域变化：清空并重载库位。
  4. 缓存规则：
     - 会话级缓存 + TTL（建议 5~10 分钟）。
     - 主数据增改停用后触发缓存失效与刷新。
  5. 提交流程：前端做必填，最终有效性以后端校验为准。
- 验收标准：
  1. 单据页主数据不允许自由文本输入。
  2. 联动正确，不出现跨仓跨区域错误。
  3. 主数据变更后可在短时间内看到刷新结果。

### [TODO] TODO-A005 大数据量主数据选择器（物料/库位）优化

- 背景：
  当物料、库位规模上千或上万时，普通下拉会出现加载慢、定位难、误选率高的问题。
- 目标：
  将“简单下拉”升级为“可搜索、可分页、可联动、可回显”的选择器，保证单据录入效率。
- 待办：
  1. 选项接口升级为“服务端搜索 + 分页”模式（关键词、状态、仓库/区域过滤）。
  2. 前端输入采用防抖搜索（建议 300ms），不再一次性加载全部选项。
  3. 选择器支持滚动分页（或虚拟列表），保证大数据量流畅渲染。
  4. 联动策略：
     - 先选仓库/区域，再查库位；
     - 物料按“编码+名称”双字段检索。
  5. 展示策略：
     - 列表显示“编码 + 名称 + 关键补充信息（如规格/单位）”；
     - 已选项保留完整回显，避免只显示 ID。
  6. 兜底方案：
     - 提供“高级选择弹窗”（表格检索）用于复杂筛选场景；
     - 支持扫码枪输入编码后自动定位选项（后续可选）。
  7. 缓存策略：
     - 热门关键词/最近使用项做短时缓存；
     - 主数据变更后失效相关缓存。
- 验收标准：
  1. 千级以上数据量下，选择器可在可接受时间内完成搜索与选择。
  2. 单据录入过程不再依赖全量下拉加载。
  3. 已有单据回显正确，不出现“只显示 ID 不显示名称”。
  4. 与仓库/区域联动规则一致，不出现跨仓错误选项。
